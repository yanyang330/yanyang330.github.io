# Jiaxuan Zou Homepage Design Specification

本文档定义本站的字体设计、视觉审美和落地规则。目标是让首页、博客、论文列表、项目页、CV 页和通用组件保持同一种学术个人站气质。

## 1. 设计方向

**定位**：研究型个人主页，优先服务学术读者、合作者、招生/实习评审者和长期阅读博客的访问者。

**审美关键词**：

- 学术编辑感：像一份精心排版的研究简历，而不是营销落地页。
- 安静、清晰、可信：信息密度可以高，但视觉噪声要低。
- 轻微个性化：使用接近 Claude 的陶土橙作为识别色，避免大面积渐变、装饰球、过强阴影。
- 内容优先：头像、联系方式、论文、博客标题和正文可读性高于装饰性。

## 2. 字体系统

### 字体族

站点使用 Anthropic 公开替代方案中的 serif + sans 双字体系统：

- 标题字体：`Poppins`
- 正文字体：`Lora`
- 代码字体：`SFMono-Regular`, `Cascadia Code`, `Fira Code`, `Menlo`, `Consolas`
- 中文标题 fallback：`Songti SC`, `STSong`, `SimSun`
- 中文正文 fallback：`PingFang SC`, `Hiragino Sans GB`, `Microsoft YaHei`

使用规则：

- `h1` 到 `h6` 使用 `Poppins`，形成接近 Anthropic 的干净现代标题感。
- 正文段落使用 `Lora`，保留温暖、书面、可阅读的质感。
- 导航、按钮、元信息、表头使用 `Poppins`，保证扫描效率。
- 代码、键盘快捷键、行内代码使用 mono。
- 不新增一次性 `font-family`。如需新字体，先更新 `_sass/_variables.scss` 和本规范。

### 字号阶梯

字号使用静态 token，不根据视口宽度缩放：

| Token         |       Size | 用途                             |
| ------------- | ---------: | -------------------------------- |
| `--text-xs`   |  `0.75rem` | 页脚、注释、徽标、小型辅助信息   |
| `--text-sm`   | `0.875rem` | 元信息、联系方式说明、表格、代码 |
| `--text-base` |     `1rem` | 正文、列表、普通按钮             |
| `--text-lg`   | `1.125rem` | 小节强调、卡片标题的低层级场景   |
| `--text-xl`   | `1.375rem` | `h3`、模块内重点标题             |
| `--text-2xl`  |  `1.75rem` | `h2` 或大型图标组                |
| `--text-3xl`  | `2.125rem` | 页面内主标题                     |
| `--text-4xl`  | `2.625rem` | 主页姓名、重要页面标题           |
| `--text-5xl`  |  `3.25rem` | 极少数展示性元素                 |

### 行高与字距

| Token               |  Value | 用途                   |
| ------------------- | -----: | ---------------------- |
| `--leading-tight`   | `1.15` | 一级标题               |
| `--leading-title`   | `1.25` | 标题、短标签           |
| `--leading-normal`  |  `1.6` | 常规正文               |
| `--leading-relaxed` | `1.75` | 博客正文、长段落、引用 |

字距规则：

- 全站 `letter-spacing` 保持 `0`。
- 不使用负字距压缩标题。
- 大写标签通过字重、边框和间距表达层级，不靠字距制造风格。

## 3. 颜色系统

基础色：

- 背景：`#faf9f5`
- 卡片背景：`#ffffff`
- 文字：`#141413`
- 弱文字：`#73726c`
- 边框：`rgba(31, 30, 29, 0.15)`
- 品牌色：`#d97757`
- 深色背景：`#30302e`
- 深色卡片背景：`#262624`
- 深色文字：`#faf9f5`
- 深色弱文字：`#c2c0b6`

使用规则：

- 品牌色只用于链接、当前导航项、重要引用边线、少量徽标。
- 正文不使用品牌色强调整段内容。
- 论文列表和 CV 页的按钮保持线框样式，悬停时使用品牌色。
- 页面背景保持 Claude 式暖白；卡片和局部承载面可以使用白色。

## 4. 版式与布局

最大内容宽度由 `_config.yml` 的 `max_width` 控制，当前为 `930px`。

版式原则：

- About 首页采用左右信息流：左侧文字，右侧头像和联系入口。
- 联系方式放在 headshot 下方，不放在页面底部作为大图标块。
- 长正文段落最大宽度控制在约 `72ch`，保持舒适阅读长度。
- 页面区块不做大面积卡片化。卡片只用于重复项、CV 分组、项目列表等真实容器。
- 标题上方留足呼吸空间，标题下方紧跟相关内容。

## 5. 组件规范

### 导航

- 字体使用 sans。
- 当前页面导航项使用 `font-weight: 600` 和品牌色。
- 搜索、主题切换、语言切换保持紧凑，不挤占主导航。

### 头像与联系方式

- 头像保持圆形，阴影轻微。
- 机构信息使用 `--text-sm` 居中。
- 联系图标使用紧凑尺寸，置于头像说明下方。
- 联系说明使用 `--text-xs`，避免抢正文层级。

### 论文列表

- 标题使用正文色和 `font-weight: 600`。
- 作者、venue、按钮遵循正文 sans 系统。
- `BIB`, `PDF`, `VIDEO` 等按钮保持线框和小尺寸，不做高饱和填充。

### 博客

- 列表页标题可使用 serif 和品牌色。
- 正文使用 `--text-base` 与 `--leading-relaxed`。
- 代码块使用 `--text-sm`，移动端不继续缩小字号。

### CV

- 章节标题使用 serif，但条目内部元信息使用 sans。
- 条目副标题统一用 `.cv-entry-meta`。
- 条目注释/摘要统一用 `.cv-entry-note`。
- 不在 CV include 中新增内联 `font-size`。

## 6. 交互与动效

- 默认动效使用 `--transition-fast` 或 `--transition-base`。
- 交互反馈优先改变颜色、边框、轻微阴影。
- 不使用弹跳、夸张位移或持续循环动画。
- 焦点态必须清晰，使用品牌色 outline。

## 7. 响应式规则

- 响应式主要通过布局、间距、换行和容器宽度处理。
- 不使用 `clamp()` 或 viewport 单位缩放全局字号。
- 移动端不隐藏核心内容：导航、联系方式、论文按钮和 CV 信息都应可访问。
- 任何新增文本容器必须考虑长英文标题、中文混排和链接换行。

## 8. 实现规则

新增或修改样式时遵循以下顺序：

1. 先使用 `_sass/_variables.scss` 中的 token。
2. 再使用 `_sass/_themes.scss` 暴露的 CSS variables。
3. 只有当组件有真实独特需求时，才新增局部 class。
4. 不新增页面级内联 `style` 来控制字体、字号、颜色。
5. 不改第三方库、生成文件或 Font Awesome 内部样式。

核心文件：

- `_sass/_variables.scss`：字体、字号、行高、圆角、动效 token。
- `_sass/_themes.scss`：把 Sass token 暴露成 CSS variables。
- `_sass/_typography.scss`：全局正文、标题、链接、表格、引用、代码。
- `_sass/_components.scss`：头像、联系方式、卡片和通用组件。
- `_sass/_blog.scss`：博客和文章排版。
- `_sass/_publications.scss`：论文列表排版。
- `_sass/_cv.scss`：CV 页排版。

## 9. 检查清单

每次视觉改动后检查：

- 标题是否使用 serif，正文是否使用 sans。
- 是否新增了硬编码字体或字号。
- 是否出现负字距。
- 长标题、长链接、中文混排是否换行正常。
- About 页联系方式是否仍在头像下方。
- 明暗主题下链接、按钮、页脚对比度是否可读。
- 移动端导航、头像、论文按钮和 CV 条目没有重叠。
